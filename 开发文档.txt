# 【AI开发指令】多功能文档格式转换网站 (DocMaster) —— 项目需求文档与开发计划

## 一、 项目概览
开发一个支持主流办公文档格式（PDF、Word、PPT）之间相互转换的 Web 网站。要求功能全面、界面现代、处理速度快，且包含文件管理功能。
目标平台：浏览器 Web 端。
开发环境：VSCode，使用 Claude Code 终端（v2.x，已安装 Superpowers 等插件）。

## 二、 功能需求清单 (User Stories)
1. **格式转换核心功能**：
   - PDF 转 Word (.docx) —— 要求排版尽量还原。
   - PDF 转 PPT (.pptx) —— 将 PDF 的每一页转换为 PPT 的幻灯片。
   - Word 转 PDF —— 直接转换。
   - PPT 转 PDF —— 直接转换。
   - PPT 转 Word —— 提取 PPT 文本内容生成 Word 文档。
   - 支持 PDF 合并/拆分（作为额外增值功能）。
2. **文件管理**：
   - 支持拖拽上传文件（Drag & Drop）。
   - 显示上传进度和转换进度（可显示百分比进度条）。
   - 转换完成后，提供一键下载生成的文档。
   - 支持历史转换记录查看和清空。
3. **用户体验 (UX)**：
   - 极简风格 UI，适配移动端和 PC 端。
   - 无广告，开箱即用。
   - 异常处理（如文件过大、格式不支持、转换失败，需给出友好提示）。

## 三、 技术选型与核心依赖
本项目采用 **前后端分离** 架构。
- **前端技术栈**：React / Vue3 + Vite（推荐 React 搭配 Ant Design 或 MUI 组件库，方便快速构建 UI）。部署于 Node 环境。
- **后端技术栈**：Python + FastAPI（强推荐，因为 Python 生态拥有世界上最强大的文档处理库，如 pdf2docx, python-pptx, PyPDF2, unoconv 等）。部署于 Python 环境。
- **核心依赖库推荐**（需 AI 进行技术调研确认最新版本）：
  - `pdf2docx` (用于 PDF 转 Word)
  - `python-pptx` (用于生成 PPT)
  - `PyPDF2` 或 `pypdf` (用于 PDF 操作)
  - `libreoffice-core` / `unoconv` (用于利用 LibreOffice 完成 PPT 到 Word 等复杂转换)
  - `fastapi` + `uvicorn` (后端 WEB 框架)
  - `pydantic` (数据校验)

## 四、 后端 API 设计 (FastAPI)
提供以下接口：
1. `POST /api/upload`：接收文件，返回一个唯一的 task_id。
2. `POST /api/convert`：传入 task_id 和 转换目标格式（如 pdf_to_word, pdf_to_ppt），启动异步转换任务。
3. `GET /api/status/{task_id}`：查询转换任务进度（0% - 100%）。
4. `GET /api/download/{task_id}`：下载转换完成后的文件。
5. `DELETE /api/files/{task_id}`：清理服务端的临时文件。
